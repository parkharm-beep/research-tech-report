# 📑 공공데이터의 인공지능 친화적 관리 가이드라인

> [!abstract] **한 줄 요약 (Core Essence)**
> 공공데이터를 AI 모델이 즉각 활용 가능한 'AI-Ready Data'로 전환하기 위한 포맷·메타데이터·품질·거버넌스 기준을 공공기관 실무 담당자에게 제시한다.

---

## 1. 🔍 배경 및 목적 (Context)

- **배경:** 전 세계 AI 전환 가속화에 따라 공공데이터 관리 패러다임이 '인간 가독성 기반 개방'에서 'AI 모델이 즉각 학습·추론할 수 있는 상태'로 근본적으로 전환되고 있음
- **목적:** 공공데이터 담당자에게 데이터 전처리 비용 절감과 AI 모델 신뢰성 확보를 위한 전략적 기준 제공; 국가 AI 혁신 생태계 조성 및 글로벌 표준 정합성 확보
- **핵심 타겟:** 「공공데이터법」 상 공공기관의 데이터를 생산·관리·제공하는 실무 담당자 (정보시스템 자체는 제외)

---

## 2. 🏗️ 주요 개념 및 구조 (Key Framework)

### AI 친화적 공공데이터 정의
- **AI-Ready Data:** AI 모델의 학습·추론·평가에 즉각 투입될 수 있도록 기계 이해성과 고도화된 품질이 확보된 데이터
- 기존 '인간 가독성' 중심 → **기계 이해성(Machine Readability)** + **머신 액셔너블(Machine-Actionable)** 중심

### As-Is vs To-Be 비교

| 구분 | 기존 (As-Is) | AI 친화적 (To-Be) |
|---|---|---|
| 공개 목적 | 투명성·알 권리 | AI 학습·추론 활용 최적화 |
| 데이터 품질 | 오류 검수·정합성 | 대표성·적합성·편향 명시 |
| 데이터 구조 | CSV, XML | Parquet 등 대량 처리 최적화 |
| 접근 방식 | 개별 파일 다운로드 | 대량 다운로드·버전 관리·PID 부여 |
| 메타데이터 | 기본 속성(제목·요약) | 생성 과정·품질 지표·편향·한계 |
| 권리·라이선스 | 공공누리 표준 라이선스 | 저작권·개인정보·재사용 조건 명시 |
| 위험·윤리 | 법령·보안 준수 중심 | AI 편향·차별 방지 사전 평가 |

### 4대 핵심 특성

1. **기계 가공 및 상호운용성** — 오픈 포맷 준수 + RESTful API·PID 기반 연속성
2. **메타데이터 및 문서화** — 국제 표준(Croissant·DCAT) 적용, 생성 과정·한계 투명 공개
3. **내용 품질과 대표성** — 편향 최소화, 최신성·무결성 확보
4. **권리·윤리·보안 및 책임 거버넌스** — 라이선스 명확화, 책임 주체·연락 창구 지정

## 3. 📝 세부 실행 가이드 (Actionable Insights)

### 데이터 구조 및 접근 체계

- **오픈 포맷 적용 (필수)**
  - **권장:** CSV, JSON, **[[Parquet]]** (열 지향, GPU 환경 자동 처리 최적화), GeoJSON
  - **비권장:** HWP, DOCX, XLSX, PSD 등 (변환 비용 발생, 자동화 저해)
- **기계 이해 가능 구조:** ISO 8601(날짜), ISO 3166(국가) 등 국제 표준 형식 필수 적용
- **API 제공:** 갱신 빈번 데이터는 RESTful API + URI 기반 영구 식별자(PID); 대량 처리용 스트리밍 지원

### 메타데이터 체계 (국제 표준 준수)

- **[[Croissant]]** — AI 모델·학습 도구의 데이터 즉시 처리를 위한 표준
- **DCAT** — 플랫폼 간 검색 연계
- **Dublin Core** — 범용적 기술(description)
- **5대 메타데이터 구성:** 데이터 관리 / 데이터 계보(버전·변경 이력) / 데이터 이용(라이선스) / 데이터 품질(편향·한계점) / 타입별 정보(이미지·음성·영상 등)

### 데이터 품질 6대 지표

| 지표 | 내용 |
|---|---|
| 완전성 | 필수 항목 누락·결측치 관리 |
| 정확성 | 실제 값 일치성, AI Ground Truth 활용 가능성 |
| 유효성 | ISO 등 표준 형식·허용 범위 준수 |
| 일관성 | 데이터 간 논리적 모순 부재 |
| 적시성 | 갱신 주기 준수·실시간성 확보 |
| 유일성 | 중복 데이터 배제 |

### 보안 및 윤리 준수: 공유 데이터 자격 등급

| 등급 | 대상 | 공유 범위 |
|---|---|---|
| 1등급 (공개공유) | 개인정보·보안 문제 없는 데이터 | 전 국민 개방 |
| 2등급 (제한공유) | 비식별 조치로 안전성 확보된 데이터 | 승인 기관 간 공유 |
| 3등급 (내부전용) | 비식별 후에도 외부 공유 부적절한 데이터 | 기관 내부 한정 |
| 공유 불가 | 고유식별정보·국가보안사항·재식별 위험 잔존 | 공유 금지 |

### 가명정보 처리 5단계 ([[k-익명성]] 적용)

1. **사전준비** — 대상 데이터·처리 목적·책임자 확인
2. **위험성 검토** — 재식별 가능성 평가
3. **가명처리** — k-익명성 확보, 마스킹 등 적용
4. **적정성 검토** — 처리 결과의 재식별 위험 재평가
5. **안전한 관리** — 접근 통제·감사 추적

## 4. 📊 요약 및 결론 (Summary & Evaluation)

- **핵심 결론:** 공공데이터가 국가 AI 경쟁력을 좌우하는 핵심 인프라로 기능하려면 포맷·메타데이터·품질·보안 네 축에서 동시에 AI-Ready 전환이 필요
- **시사점:**
  1. Parquet 등 오픈 포맷 전환이 AI 전처리 비용을 직접 절감하는 핵심 수단
  2. Croissant 기반 편향·한계 명시가 모델의 사회적 신뢰성 확보로 이어짐
  3. 책임 주체·PID 기반 버전 관리가 연구 재현성 및 법적 안정성 확보에 필수
- **한계:** 기존 데이터셋에 소급 적용 시 추가 공수 발생; Croissant 생태계 성숙 중
