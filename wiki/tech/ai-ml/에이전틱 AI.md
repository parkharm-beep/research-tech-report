# 에이전틱 AI (Agentic AI)

## 1. 개요 (Overview)
- **기술명:** 에이전틱 AI (Agentic AI)
- **정의:** 사용자의 목표를 달성하기 위해 LLM이 능동적으로 계획·도구 호출·기억 유지를 반복하며 자율적으로 동작하는 AI 시스템
- **배경:** 기존 단발성 Q&A 방식의 LLM 활용에서 벗어나, 복잡한 다단계 작업을 AI가 스스로 수행하는 수요가 증가하면서 에이전트 패러다임이 주목받게 됐다.

## 2. 주요 특징 (Key Features)
- **장기 컨텍스트(기억):** 에이전트가 수행한 모든 행동·관찰·대화를 컨텍스트로 누적하여 후속 의사결정에 활용
- **도구 호출(Tool Calling):** 검색, 코드 실행, 외부 API 등을 반복 호출하며 루프 진행
- **에이전트 루프:** 계획 → 도구 호출 → 결과 관찰 → 재계획을 반복하는 순환 구조
- **CPU 병목 유발:** LLM 추론(GPU)과 달리 에이전트 오케스트레이션, 데이터 검색, 도구 호출 조정은 CPU에서 실행 → GPU 대비 CPU 부하 비율 급상승 (추정 50:50)
- **메모리 수요 폭증:** 에이전트가 누적하는 컨텍스트(외부 지식 + 에이전트 메모리 + KV 캐시)가 기하급수적으로 증가

## 3. 작동 원리 (How it Works)
에이전틱 AI의 메모리는 세 가지 계층으로 구성된다:
1. **외부 지식 (External Knowledge):** 벡터 DB, 그래프 DB 등에 저장된 장기 지식. RAG로 필요 시 검색
2. **에이전트 메모리 (Agentic Memory):** 현재 작업 세션에서 에이전트가 수행한 이력 전체. 스토리지에 저장 후 필요 시 로드
3. **워킹 메모리 (Working Memory / KV 캐시):** 현재 추론에 직접 사용되는 단기 기억. HBM에 상주

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: 코딩 에이전트 — 코드 작성 → 실행 → 오류 수정을 반복하며 전체 기능 구현
- 사례 2: 데이터 분석 에이전트 — 대규모 데이터에서 반복 검색·집계·시각화 수행
- 사례 3: 기업 문서 자동화 — 계약서 검토, 보고서 작성 등 복잡한 비즈니스 워크플로우 자동화
- 사례 4: MCP 연동 에이전트 — AI가 [[MCP]]를 통해 Figma·GitHub 등 외부 툴을 직접 조작

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 복잡한 다단계 작업을 인간 개입 없이 처리 가능
- 도구 호출로 LLM 학습 데이터 한계를 실시간으로 극복
- 상태(컨텍스트) 유지로 장기 프로젝트 수행 가능
### 단점 (Cons)
- 에이전트 루프가 돌수록 CPU 부하와 메모리 수요 기하급수적 증가
- KV 캐시 및 에이전트 메모리 관리의 복잡성
- 오류 누적 가능성 (한 단계 실수가 이후 모든 단계에 전파)
- 실행 비용이 단순 LLM 호출 대비 수배~수십 배
