# 양자화 (Quantization)

## 1. 개요 (Overview)
- **기술명:** 양자화 (Quantization)
- **정의:** LLM 모델의 가중치를 고정밀도(FP32, FP16)에서 저정밀도(INT8, INT4 등)로 변환하여 모델 크기와 메모리 사용량을 줄이는 모델 경량화 기법
- **배경:** LLM 모델 크기가 수십~수백 GB에 달하면서 일반 하드웨어에서 실행이 불가능해졌다. 양자화를 통해 모델을 수 GB 수준으로 압축하면 소비자용 GPU나 맥북에서도 추론·파인튜닝이 가능해진다.

## 2. 주요 특징 (Key Features)
- **압축률:** 원본 비트 ÷ 양자화 비트 (FP32→INT4: 8배 압축)
- 0.8B INT4 모델 ≈ 400MB → 일반 노트북 RAM에서 실행 가능
- 8B INT4 모델 ≈ 4GB → 소비자용 GPU (RTX 4070 등)에서 실행 가능
- 추론 latency: 로컬 양자화 모델이 클라우드 API 대비 10배 이상 빠를 수 있음
- Unsloth (NVIDIA GPU), MLX (Apple Silicon) 등 플랫폼별 최적화 프레임워크 지원

## 3. 작동 원리 (How it Works)
- **비트 정밀도 변환:** 각 가중치 값을 더 적은 비트로 표현
  - FP32 (32비트) → INT8 (8비트): 4배 압축
  - FP32 (32비트) → INT4 (4비트): 8배 압축
- **스케일링:** 값 범위를 유지하기 위해 양자화 전 스케일 팩터 계산
- **증류(Distillation)와 결합:** 큰 모델의 추론 과정을 학습한 소형 모델을 만들고, 해당 소형 모델을 양자화하여 SLM 생성
- VRAM 계산: 파라미터 수(B) × 비트 수 ÷ 8 = 바이트 → 학습 시 약 10배 곱하여 VRAM 추정

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: 온프레미스 보안 환경 — 외부 API 전송 불가 데이터를 로컬 양자화 모델로 처리
- 사례 2: 비용 최적화 — 월 수백만 원의 API 비용을 GPU 서버 초기 구매비용으로 대체
- 사례 3: 엣지 디바이스 배포 — 스마트폰, 노트북 등 제한된 하드웨어에서 AI 기능 구현

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 모델 크기와 메모리 사용량 대폭 감소 (4~8배)
- 추론 속도 향상 (메모리 I/O 감소 효과)
- 일반 하드웨어에서 대형 모델 실행 가능
- API 의존성 제거 → 보안, 비용, 가용성 개선
### 단점 (Cons)
- 정밀도 손실로 인한 성능 저하 (압축률이 높을수록 심화)
- 모든 태스크에서 원본 대비 성능이 다소 떨어짐
- 양자화 방식과 하드웨어에 따라 호환성 이슈 존재

> [!note] 2026-05-12 소스 추가: PTQ vs QAT 구분
> **PTQ (Post-Training Quantization):** 학습 완료 후 양자화 적용. 빠르고 쉽지만 GPU 종류·모델에 따라 성능 저하 심할 수 있음. **QAT (Quantization-Aware Training):** 학습 단계에서 양자화 적용. 성능 유지가 더 좋지만 복잡. **실무 표준:** 8비트(INT8)이 가장 보편적. 4비트는 손실 크므로 결과 확인 후 사용. FP32→INT8 변환 시 정밀도는 낮아지지만 4배 압축 효과. (출처: LG전자 AI연구소 박사 강의)
