# 희소 어텐션 (Sparse Attention)

## 1. 개요 (Overview)
- **기술명:** 희소 어텐션 (Sparse Attention)
- **정의:** 트랜스포머의 어텐션 메커니즘에서 모든 토큰 쌍을 비교하는 대신 의미적으로 중요한 일부 관계만 선택적으로 계산하여 연산량을 선형(linear) 수준으로 줄이는 아키텍처 기법
- **배경:** 기존 트랜스포머는 모든 토큰을 서로 비교하는 풀 어텐션(full attention) 구조로, 입력 길이가 두 배 늘면 연산량이 네 배 증가하는 '쿼드래틱(quadratic) 스케일링' 문제가 발생한다. 이로 인해 산업 전반에서 긴 컨텍스트 처리에 제약이 컸고, 이를 우회하기 위한 RAG·청킹 등의 복잡한 워크어라운드가 필요했다.

## 2. 주요 특징 (Key Features)
- 전체 토큰 쌍 비교 대신 의미적으로 중요한 관계만 선택 계산
- 입력 길이 증가에도 연산량이 **선형(O(n))** 으로 증가 (기존 풀 어텐션: O(n²))
- 동일 연산량 기준 처리 가능한 컨텍스트 길이가 대폭 확장
- KV 캐시 메모리 요구량 획기적 절감 (기존 대비 수백분의 1 수준 사례 존재)

## 3. 작동 원리 (How it Works)
- 입력 토큰 전체 중 의미 관련성이 높은 일부 토큰 쌍만 어텐션 연산 수행
- '어떤 쌍을 계산할지' 결정하는 선택 메커니즘이 핵심 (로컬 윈도우·글로벌 토큰·학습 기반 선택 등 다양한 전략 존재)
- 서브쿼드래틱(Subquadratic) 아키텍처: 희소 어텐션 기반으로 1200만 토큰 컨텍스트 창 구현 사례 (SubQ 1M-Preview, 2026-05-06 공개)
  - 장문 추론 벤치마크 약 95% 정확도, 동일 작업 비용 수백 배 절감 주장 (독립 검증 진행 중)

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: 초장문 문서 처리 — 수백만~수천만 토큰을 단일 맥락으로 이해 (코드베이스, 대규모 DB, 장기 대화 이력)
- 사례 2: RAG 우회 — 문서를 청킹하지 않고 전문을 직접 입력해 검색 시스템 의존도 감소

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 초장문 컨텍스트를 낮은 비용으로 처리 가능
- KV 캐시 메모리 압박 해소 → GPU 비용 절감
- RAG·청킹 등 복잡한 워크어라운드 시스템을 단순화할 가능성

### 단점 (Cons)
- 어떤 토큰 쌍이 '중요한지' 결정하는 선택 메커니즘의 품질에 성능이 의존
- 풀 어텐션 대비 일부 관계 누락 가능성 (정확도 트레이드오프)
- 상용 수준의 독립 벤치마크 검증 사례가 아직 제한적
