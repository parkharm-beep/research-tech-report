# δ-mem (Delta-mem)

## 1. 개요 (Overview)
- **기술명:** δ-mem (Delta-mem, Efficient Online Memory for LLMs)
- **정의:** 동결된(frozen) 풀어텐션 LLM 백본에 고정 크기의 연상 메모리(associative memory) 상태를 부착하여, 컨텍스트 창 확장 없이 장기 정보를 압축·재사용하는 경량 온라인 메모리 메커니즘
- **배경:** LLM 기반 장기 어시스턴트·에이전트 시스템은 대화 이력을 누적하고 재사용해야 한다. 그러나 단순 컨텍스트 창 확장은 비용이 크고([[KV 캐시]] 선형 증가), 효과적인 컨텍스트 활용을 보장하지 못한다. δ-mem은 백본 교체·파인튜닝 없이 이를 해결한다.

## 2. 핵심 아이디어

### 문제: KV 캐시의 선형 증가
- 컨텍스트 토큰 수 증가 → KV 캐시 메모리 선형 비례 증가
- 장기 대화·에이전트 작업에서 HBM 압박 심화

### 해결: 고정 크기 온라인 메모리 상태
- 과거 정보를 **고정 크기 상태 행렬(state matrix)**로 압축
- 새 토큰 입력 시 델타 규칙(delta-rule) 학습으로 상태를 온라인 업데이트
- 상태의 readout이 백본 어텐션 계산에 **저랭크 보정(low-rank correction)**을 생성

## 3. 작동 원리 (How it Works)

```
입력 토큰 시퀀스
  ↓
[동결된 풀어텐션 백본]
  + [δ-mem 온라인 상태 모듈]
      ├─ delta-rule로 상태 행렬 업데이트 (과거 정보 압축)
      └─ readout → 어텐션 계산에 저랭크 보정 적용
  ↓
생성 토큰
```

1. **상태 압축:** 각 입력 토큰을 처리하면서 고정 크기 연상 메모리 행렬에 정보 누적
2. **Delta-rule 업데이트:** 새 정보와 기존 상태 간의 오차(δ)를 기반으로 온라인 학습
3. **어텐션 보정:** 메모리 readout으로 백본의 어텐션 Key/Value에 저랭크 보정 적용
4. **백본 동결:** 기존 LLM 가중치 변경 없음 → 일반 능력 보존

## 4. 주요 특징 (Key Features)
- **고정 메모리 예산:** 컨텍스트 길이에 무관한 상수 크기 상태 → KV 캐시 선형 증가 문제 회피
- **백본 무수정:** 동결된 풀어텐션 LLM에 부착 → 기존 모델 재활용 가능
- **파인튜닝 불필요:** δ-mem 모듈만 경량 학습, 백본 전체 파인튜닝 불필요
- **컨텍스트 확장 불필요:** 명시적 컨텍스트 창 확장 없이 장기 기억 구현
- **어텐션 직접 결합:** 메모리가 어텐션 계산과 직접 연결 → 정보 활용 효율 향상

## 5. 성능 결과 (Benchmarks)
- **MemoryAgentBench:** 메모리 집약 에이전트 벤치마크에서 큰 성능 향상
- **LoCoMo:** 장기 대화 메모리 벤치마크에서 향상
- **일반 능력:** 기존 백본 수준 유지 (일반 태스크 성능 저하 최소화)
- 비교군: 동결 백본 단독, 기존 메모리 베이스라인 중 최강 모델 대비 개선

## 6. 주요 활용 사례 (Use Cases)
- 사례 1: 장기 어시스턴트 — 수백 턴의 대화 이력을 고정 메모리에 압축해 참조
- 사례 2: 에이전트 시스템 — 다단계 작업에서 이전 관찰·행동 이력을 효율적으로 기억
- 사례 3: 긴 문서 처리 — 컨텍스트 창 한계 없이 문서 전체를 메모리 상태에 요약

## 7. 장단점 (Pros & Cons)
### 장점 (Pros)
- KV 캐시와 달리 메모리 사용량이 컨텍스트 길이에 무관하게 일정
- 기존 LLM에 플러그인 방식으로 적용 가능 (백본 교체 불필요)
- 전체 파인튜닝 대비 훨씬 낮은 학습 비용
### 단점 (Cons)
- 고정 크기 상태로 압축 시 정보 손실 발생 가능
- Delta-rule 기반 온라인 학습의 안정성 및 망각 문제
- 극단적으로 긴 시퀀스에서 압축 한계에 도달할 수 있음
