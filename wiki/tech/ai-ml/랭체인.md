# 랭체인 (LangChain)

## 1. 개요 (Overview)
- **기술명:** 랭체인 (LangChain)
- **정의:** 대규모 언어 모델(LLM)을 활용한 애플리케이션 개발을 간소화하고 가속화하기 위한 오픈소스 프레임워크입니다.
- **배경:** LLM을 단순히 API로 호출하는 것을 넘어, 데이터 소스와의 연결, 문맥 유지(Memory), 여러 단계를 거치는 복잡한 워크플로우(Chains)를 효율적으로 관리하고 구축하기 위해 등장했습니다.

## 2. 주요 특징 (Key Features)
- **모듈성 (Modularity):** 모델(LLM/ChatModels), 프롬프트 템플릿, 인덱스(Vector Store), 메모리, 에이전트 등 다양한 컴포넌트를 독립적으로 제공합니다.
- **체이닝 (Chaining):** LCEL(LangChain Expression Language)을 통해 여러 작업을 파이프라인 형태로 연결하여 복잡한 로직을 구성할 수 있습니다.
- **데이터 연결 (Data Connection):** 외부 데이터 소스(PDF, 웹사이트, 데이터베이스 등)를 로드하고 가공하여 LLM에 전달하는 기능을 제공합니다.
- **에이전트 (Agents):** 어떤 도구를 사용할지 LLM이 스스로 판단하여 작업을 수행하도록 하는 지능형 에이전트 구축이 가능합니다.

## 3. 작동 원리 (How it Works)
- 사용자의 입력을 받으면 프롬프트 템플릿을 통해 최적화된 명령어로 변환합니다.
- 필요한 경우 외부 데이터베이스(Vector Store)에서 관련 정보를 검색(Retrieval)하여 컨텍스트에 추가합니다.
- 구성된 '체인(Chain)'을 따라 LLM을 호출하고, 결과값을 파싱하여 최종 응답을 생성하거나 다음 단계로 전달합니다.

## 4. 주요 활용 사례 (Use Cases)
- **RAG (Retrieval-Augmented Generation):** 사내 문서나 특정 지식 베이스를 기반으로 답변하는 챗봇.
- **데이터 분석 및 요약:** 대량의 텍스트 데이터를 요약하거나 특정 정보를 추출하는 자동화 파이프라인.
- **개인화 비서:** 사용자의 과거 대화 내용을 기억(Memory)하고 맞춤형 정보를 제공하는 비서 서비스.

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 방대한 생태계와 커뮤니티 지원으로 참고할 자료가 많습니다.
- 다양한 LLM 제공업체 및 외부 도구와의 통합이 매우 용이합니다.
- 복잡한 LLM 워크플로우를 표준화된 방식으로 개발하여 유지보수성을 높일 수 있습니다.

### 단점 (Cons)
- 추상화 계층이 깊어 내부 동작을 완벽히 이해하거나 세밀하게 디버깅하기 어려울 수 있습니다.
- 라이브러리 업데이트가 매우 빨라 버전 간 하위 호환성 이슈가 발생할 수 있습니다.
- 매우 복잡한 순환 구조(Cycle)나 상태 관리가 필요한 경우 랭그래프(LangGraph) 같은 추가 도구가 필요합니다.
