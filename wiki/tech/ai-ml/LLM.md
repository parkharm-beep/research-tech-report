# LLM (Large Language Model)

## 1. 개요 (Overview)
- **기술명:** LLM (Large Language Model, 대규모 언어 모델)
- **정의:** 수천억 개의 파라미터를 가진 트랜스포머 기반 신경망으로, 대규모 텍스트 코퍼스를 학습하여 자연어 이해·생성·추론 등 다양한 언어 태스크를 수행하는 AI 모델
- **배경:** 2017년 트랜스포머 아키텍처 등장 이후 GPT·BERT 시리즈를 거쳐 GPT-4·Claude·Gemini 등 초대형 모델로 발전했다.

## 2. 주요 특징 (Key Features)
- **사전학습 (Pre-training):** 대규모 텍스트에서 다음 토큰 예측(Next Token Prediction)으로 자율 학습
- **파인튜닝:** 특정 태스크·도메인에 맞게 추가 학습 — [[LoRA]], [[양자화]] 등 경량 기법 활용
- **RLHF:** 인간 피드백 강화학습으로 안전성·유용성 정렬
- **인컨텍스트 러닝:** 프롬프트 내 예시만으로 Zero/Few-shot 태스크 수행
- **창발적 능력:** 특정 규모 이상에서 논리 추론·수학 문제 풀기 등 예상치 못한 능력 출현

## 3. 작동 원리 (How it Works)
- **토크나이저:** 텍스트를 서브워드 단위(토큰)로 분할
- **트랜스포머:** Self-Attention 메커니즘으로 토큰 간 관계 포착 → [[KV 캐시]]에 저장
- **디코딩:** 이전 토큰들을 조건으로 다음 토큰의 확률 분포 계산 후 샘플링으로 생성
- **컨텍스트 윈도우:** 한 번에 처리할 수 있는 최대 토큰 수 (현재 1M+ 확장 중 → [[희소 어텐션]])

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: 텍스트 생성·요약·번역 (ChatGPT, Claude 등)
- 사례 2: 코드 생성 및 디버깅 (GitHub Copilot, Claude Code)
- 사례 3: 문서 기반 Q&A ([[RAG]])
- 사례 4: 자율 에이전트 오케스트레이션 ([[에이전틱 AI]])

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 범용성 — 단일 모델로 수백 가지 태스크 처리
- 자연어 인터페이스로 전문 지식 없이도 활용 가능
### 단점 (Cons)
- 할루시네이션 — 그럴듯하지만 틀린 정보 생성 위험
- GPU·HBM 자원 집약, 서빙 비용 높음
- 지식 컷오프 — 학습 이후 사건은 알지 못함
