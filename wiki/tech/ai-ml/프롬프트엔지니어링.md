# 프롬프트 엔지니어링 (Prompt Engineering)

## 1. 개요 (Overview)
- **기술명:** 프롬프트 엔지니어링 (Prompt Engineering)
- **정의:** LLM에게 원하는 출력을 얻기 위해 입력(프롬프트)을 체계적으로 설계·최적화하는 기술 분야
- **배경:** LLM은 동일한 지식을 갖더라도 프롬프트 구성에 따라 출력 품질이 크게 달라진다. 파인튜닝 없이 프롬프트 조정만으로 성능을 향상시킬 수 있어 빠르게 실용 기술로 자리잡았다.

## 2. 주요 기법 (Key Techniques)
- **Zero-shot:** 예시 없이 지시만으로 태스크 수행 요청
- **Few-shot:** 입력-출력 예시 몇 가지를 제공하여 패턴 학습 유도
- **Chain-of-Thought (CoT):** "단계별로 생각해보자"로 논리적 추론 유도 → 복잡한 문제에 효과적
- **Role Prompting:** "당신은 전문 법률가입니다"처럼 역할을 부여하여 전문성 향상
- **[[출력 계약]]:** JSON 스키마 등 출력 형식을 명시적으로 지정
- **구조화된 프롬프트:** XML·마크다운 헤더로 시스템/사용자/컨텍스트 구간 명확히 구분

## 3. 작동 원리 (How it Works)
- LLM은 입력 토큰 시퀀스 전체를 컨텍스트로 해석
- 프롬프트의 구조·어조·예시·제약이 다음 토큰 예측 확률 분포에 직접 영향
- "Think step by step" → 중간 추론 단계를 토큰으로 생성하게 하여 정확도 향상

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: Few-shot으로 특정 형식의 데이터 추출 (파인튜닝 없이)
- 사례 2: CoT로 수학 문제·논리 추론 정확도 향상
- 사례 3: 시스템 프롬프트로 챗봇 페르소나·안전 지침 설정

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 파인튜닝 없이 빠르게 모델 동작 커스터마이징 가능
- 모델 지식을 바꾸지 않고 추론 방식만 유도
### 단점 (Cons)
- 복잡한 태스크는 프롬프트만으로 한계 존재
- 프롬프트 변경에 따른 성능 예측이 어려움 (brittleness)
- 장문 프롬프트는 비용·지연 증가
