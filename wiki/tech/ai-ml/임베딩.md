# 임베딩 (Embedding)

## 1. 개요 (Overview)
- **기술명:** 임베딩 (Embedding)
- **정의:** 텍스트·이미지 등 비정형 데이터를 의미적 유사성이 보존된 고차원 실수 벡터로 변환하는 표현 학습 기법
- **배경:** One-hot 인코딩의 한계를 극복하기 위해 Word2Vec(2013)이 등장했고, 이후 BERT·Sentence-BERT 등 문장 수준 임베딩으로 발전했다.

## 2. 주요 특징 (Key Features)
- **의미적 유사성 보존:** 비슷한 의미의 텍스트는 벡터 공간에서 가까운 위치에 배치
- **다양한 모달리티:** 텍스트 외 이미지(CLIP)·코드·오디오도 동일 벡터 공간으로 표현 가능
- **차원 수:** 일반적으로 128~4096 차원
- **두 가지 유형:** [[Dense 임베딩]](의미 유사도), [[Sparse 임베딩]](키워드 매칭)

## 3. 작동 원리 (How it Works)
1. 텍스트를 토크나이저로 분할 후 트랜스포머 인코더에 입력
2. [CLS] 토큰 벡터 또는 평균 풀링으로 문장 벡터 추출
3. Contrastive Learning으로 유사 문서는 가깝게, 비유사 문서는 멀게 학습
4. 생성된 벡터를 [[Vector Database]]에 저장 → [[Vector Search]]로 유사도 검색

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: RAG 파이프라인 — 문서 임베딩 후 쿼리와 유사도 비교로 관련 문서 검색
- 사례 2: 시맨틱 검색 — 키워드 불일치에도 의미 기반 결과 반환
- 사례 3: 추천 시스템, 이상 감지

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 도메인·언어 무관하게 의미 기반 검색 가능
- 한 번 임베딩하면 다양한 태스크에 재사용 가능
### 단점 (Cons)
- 임베딩 생성에 GPU 또는 API 비용 필요
- 전문 용어 처리에 도메인 특화 임베딩 모델 필요
