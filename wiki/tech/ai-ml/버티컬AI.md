# 버티컬 AI (Vertical AI)

## 1. 개요 (Overview)
- **기술명:** 버티컬 AI (Vertical AI)
- **정의:** 특정 도메인의 전체 워크플로우(end-to-end pipeline)를 커버하도록 설계된 도메인 특화 AI 시스템. 범용 AI와 달리 단일 문제가 아닌 특정 산업·분야의 목적 달성 전 과정을 지원한다.
- **배경:** 범용 AI나 단일 태스크 AI는 복잡한 도메인(예: 신약 개발) 전체를 커버하지 못한다. AlphaFold가 단백질 구조 예측이라는 한 부분만 담당하듯, 목표 달성까지 이어지는 수십 개의 세부 단계에 각각 특화된 AI 엔진을 연결하는 파이프라인 접근이 등장했다.

## 2. 주요 특징 (Key Features)
- 도메인 전체 목적(goal)을 중심으로 다수의 특화 AI 엔진을 순차·병렬로 연결
- 각 엔진은 독립적으로 개선·교체 가능 (모듈형 파이프라인)
- 범용 AI로는 해결하기 어려운 도메인별 희귀 데이터·전문 지식을 직접 활용
- 신약 개발, 바이오 빅데이터 분석 등 생명과학 분야에서 선도적으로 도입

## 3. 작동 원리 (How it Works)
- **파이프라인 구성:** 도메인의 목적(예: 고혈압 신약 후보물질 발굴)을 정의한 뒤, 그 과정에 필요한 세부 문제(타깃 단백질 구조 예측 → 화합물 결합 예측 → 기능 활성화 여부 예측 → 독성 스크리닝 등)마다 특화 AI를 배치
- **데이터 레이어:** 바이오 빅데이터(단백질, 화합물, 유전체, 임상 데이터 등 16+ 카테고리)를 통합 관리하고 데이터 간 관계를 AI가 추론
- **설명 가능성(XAI):** 단순 예측 결과뿐 아니라 "왜 이 후보가 선택됐는지" 근거를 함께 제공해 전문가 의사결정을 보조
- **전문가 협업:** AI가 데이터 기반 가설을 제시하고 전문가가 검토·실험 설계 → 결과가 다시 데이터베이스에 피드백되는 반복 루프

## 4. 주요 활용 사례 (Use Cases)
- **신약 개발 파이프라인:** AlphaFold(구조 예측) + 기능 예측 AI + 바이오빅데이터 분석 플랫폼을 연결해 후보 물질 탐색부터 임상 전 단계까지 커버 (AlphaFold 4 목표, 인포보스 플랫폼 등)
- **치주 질환 후보 물질 발굴:** 기존에 언급되지 않은 화합물 9개 테스트 중 5개가 효능 확인 (인포보스 사례)
- **희귀 메커니즘 발굴:** 알려진 타깃이 아닌 경로(B/C 메커니즘)를 바이오빅데이터에서 발굴해 신규 신약 타깃 제시

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 도메인 전체 목표에 최적화 → 실험 횟수 감소 및 연구 속도 향상
- 전문가가 놓치는 데이터 관계를 대규모 데이터 처리로 보완
- 모듈형 구조로 각 AI 엔진을 독립적으로 개선 가능

### 단점 (Cons)
- 도메인마다 파이프라인을 별도로 설계해야 하므로 범용화 어려움
- 각 엔진의 오류가 파이프라인 전체에 전파될 위험
- 생명과학 분야는 외삽(extrapolation) 비율이 99%+로 예측 신뢰도 한계 상존
- 고품질 도메인 데이터 확보 및 데이터 간 관계 정의에 막대한 사전 투자 필요
