# 바이브코딩 (Vibe Coding)

## 1. 개요 (Overview)
- **기술명:** 바이브코딩 (Vibe Coding) / AI 보조 개발
- **정의:** 코딩 지식이 없는 비개발자가 Claude Code, Codex 등 AI 코딩 에이전트에게 자연어로 요구사항을 전달하여 소프트웨어를 구축하는 개발 방식
- **배경:** LLM 기반 코딩 에이전트(Claude, GPT-4o 등)가 고도화되면서 프로그래밍 언어 문법을 몰라도 원하는 기능을 자연어로 지시해 실제 동작하는 코드를 얻을 수 있게 됐다.

## 2. 주요 특징 (Key Features)
- **자연어 지시:** 코드를 직접 작성하지 않고 "SNS 메인 화면 만들어줘"처럼 의도를 말로 전달
- **AI가 코딩·디버깅 담당:** 에러 메시지를 그대로 AI에 붙여넣으면 AI가 해결책 제시
- **컨텍스트 품질이 결과 품질을 결정:** 충분한 디자인 레퍼런스, 색상 기준, 기획서를 제공할수록 출력물 품질이 높아짐 (= [[하네스엔지니어링]] 원칙)
- **MCP로 외부 툴 연동:** [[MCP]]를 활용하면 AI가 Figma, 브라우저 등을 직접 조작 가능

## 3. 작동 원리 (How it Works)
1. 비개발자가 원하는 기능·화면을 자연어로 설명
2. AI 코딩 에이전트(Claude Code, Codex 등)가 코드 생성
3. 터미널/IDE에서 코드 실행 → 에러 발생 시 에러 메시지를 AI에게 전달
4. AI가 에러 원인 분석 및 수정 코드 제공
5. 반복하며 기능 완성

## 4. 필요한 선수 지식
바이브코딩을 하려면 최소한의 개념 이해가 필요하다:
- **터미널 (Terminal):** 코드를 실행하는 커맨드라인 환경 (Windows: PowerShell, Mac: 터미널)
- **IDE (VS Code 등):** 코드 에디터 — "예쁜 메모장"
- **프레임워크/라이브러리:** 어떤 것들이 존재하는지 알아야 올바른 설계 가능 ("침실에는 침대")
- **[[Docker]]:** 작성한 코드를 어디서든 동일하게 실행하는 컨테이너 환경
- **기획·설계:** 무엇을 만들지 먼저 정의해야 AI가 정확히 구현 가능

## 5. 주요 활용 사례 (Use Cases)
- 사례 1: 비개발자가 SNS 애플리케이션(회원가입, 피드, DM) 프로토타입 제작
- 사례 2: Claude + [[MCP]] + Figma 연동으로 AI가 자동으로 UI 디자인 초안 생성
- 사례 3: 기획자·마케터가 데이터 분석 자동화 스크립트를 AI로 제작

## 6. 장단점 (Pros & Cons)
### 장점 (Pros)
- 코딩 지식 없이도 소프트웨어 프로토타입 제작 가능
- 아이디어 검증 속도 대폭 향상
- AI가 반복적인 보일러플레이트 코드를 담당
### 단점 (Cons)
- 생성된 코드의 보안·성능 이슈를 검토할 개발 지식이 없으면 위험
- 컨텍스트(기획·디자인)가 부실하면 결과물 품질이 낮음
- 복잡한 아키텍처 설계는 여전히 전문 개발자 필요
