# 랭그래프 (LangGraph)

## 1. 개요 (Overview)
- **기술명:** 랭그래프 (LangGraph)
- **정의:** LLM을 활용하여 상태가 유지되는(stateful) 다중 에이전트 애플리케이션을 구축하기 위한 라이브러리입니다.
- **배경:** 기존의 랭체인(LangChain)은 선형적인 체인이나 방향성 비순환 그래프(DAG) 구조에는 적합했으나, 에이전트가 결과에 따라 이전 단계로 돌아가거나 반복하는 '루프(Cycle)'가 포함된 복잡한 워크플로우를 구현하고 상태를 정밀하게 관리하기 위해 등장했습니다.

## 2. 주요 특징 (Key Features)
- **Cycles (루프 지원):** 그래프 구조를 통해 단계 간의 반복 및 순환 구조를 쉽게 정의할 수 있습니다.
- **Persistence (상태 보존):** 체크포인트 기능을 통해 대화의 상태를 저장하고, 중단된 시점부터 다시 시작하거나 과거 시점으로 되돌릴 수 있습니다.
- **Human-in-the-loop:** 에이전트의 동작 사이에 사람의 승인이나 개입을 넣을 수 있는 워크플로우 제어 기능을 제공합니다.
- **Fine-grained Control:** 기존의 추상화된 에이전트보다 노드와 엣지 단위로 더 세밀한 제어가 가능합니다.

## 3. 작동 원리 (How it Works)
- **State (상태):** 그래프를 관통하는 공유 데이터 구조를 정의합니다.
- **Nodes (노드):** 특정 작업(LLM 호출, 도구 사용 등)을 수행하는 함수입니다.
- **Edges (엣지):** 노드 간의 이동 경로를 정의하며, 조건에 따라 경로를 결정하는 '조건부 엣지'를 포함합니다.
- **Compilation:** 정의된 그래프를 컴파일하여 실행 가능한 애플리케이션으로 변환합니다.

## 4. 주요 활용 사례 (Use Cases)
- **사례 1: 복잡한 RAG 시스템:** 검색 결과가 부적절할 경우 다시 검색하거나 쿼리를 수정하는 피드백 루프가 포함된 RAG.
- **사례 2: 다중 에이전트 협업:** 기획, 코딩, 검수 등 역할을 나눈 에이전트들이 서로 소통하며 결과물을 완성하는 시스템.

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 복잡한 에이전트 로직을 구조화하여 관리하기 용이합니다.
- 상태 관리와 지속성 구현이 매우 강력합니다.
- LangChain 생태계와 완벽하게 통합됩니다.
### 단점 (Cons)
- 단순한 체인 구현에 비해 코드 복잡도가 높고 보일러플레이트가 많습니다.
- 학습 곡선이 존재합니다.
