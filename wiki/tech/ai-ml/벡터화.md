# 벡터화 (Vectorization)

## 1. 개요 (Overview)
- **기술명:** 벡터화 (Vectorization)
- **정의:** 텍스트·이미지·오디오 등 비정형 데이터를 수치 벡터로 변환하는 과정의 총칭으로, [[임베딩]] 생성의 전반적인 프로세스를 포함한다
- **배경:** 머신러닝 모델은 수치 입력만 처리할 수 있으므로, 원시 데이터를 모델이 이해할 수 있는 수치 표현으로 변환하는 벡터화가 모든 AI 파이프라인의 전처리 단계로 필수다.

## 2. 주요 특징 (Key Features)
- **표현 방식에 따른 분류:**
  - **One-hot 인코딩:** 단어를 어휘 크기의 이진 벡터로 표현 (의미 없음, 차원 폭발)
  - **TF-IDF:** 단어 빈도·역문서 빈도 기반 가중치 벡터 ([[Sparse 임베딩]]의 기반)
  - **Word2Vec/GloVe:** 단어 수준 밀집 벡터 (의미적 유사성 보존)
  - **Sentence Embedding:** 문장·문서 수준 밀집 벡터 ([[Dense 임베딩]])
- **다양한 모달리티:** 텍스트, 이미지(CNN/ViT), 오디오, 그래프 등 모두 벡터로 변환 가능

## 3. 작동 원리 (How it Works)
- 입력 데이터를 토큰/픽셀/프레임 단위로 분할
- 학습된 인코더 모델을 통해 고차원 실수 벡터로 변환
- 결과 벡터를 [[Vector Database]]에 저장하거나 다운스트림 모델에 입력

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: RAG 파이프라인의 문서 벡터화 → [[Vector Search]]로 검색
- 사례 2: 이미지 벡터화 → 유사 이미지 검색
- 사례 3: 로그 이벤트 벡터화 → 이상 탐지

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 모달리티에 상관없이 통일된 수치 표현으로 통합 처리 가능
### 단점 (Cons)
- 변환 과정에서 정보 손실 가능
- 고품질 벡터화에는 대규모 학습 데이터와 모델 필요
