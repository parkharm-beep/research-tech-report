# MoE (Mixture of Experts)

## 1. 개요 (Overview)
- **기술명:** MoE (Mixture of Experts, 전문가 혼합)
- **정의:** 하나의 LLM 내에 여러 개의 "전문가(Expert)" 서브네트워크를 두고, 입력에 따라 게이팅 함수가 일부 전문가만 활성화하여 추론하는 희소 활성화 아키텍처
- **배경:** 모델 전체를 매 토큰마다 활성화하면 파라미터가 늘수록 연산이 선형 증가한다. MoE는 전체 파라미터를 메모리에 올려두되 실제 연산은 소수 전문가만 수행하여, 파라미터 증가 대비 연산 비용 증가를 억제한다.

## 2. 주요 특징 (Key Features)
- **전문가(Expert):** 각 레이어에 배치된 독립적인 FFN(Feed-Forward Network) 서브네트워크
- **게이팅 함수(Gating Function):** 입력 의미를 분석하여 Top-K 전문가를 선택 (시맨틱 라우터 역할)
- **희소 활성화(Sparse Activation):** 전체 전문가 중 Top-K(보통 2~3개)만 활성화 → 활성률 ≈ 25%
- GPT-4 추정치: 전문가 8개 중 2개만 활성화 (Top-2 MoE)
- 구동에 필요한 VRAM은 여전히 전체 파라미터 기준 (전문가 전체가 메모리에 상주)
- 추론 속도 향상: 활성 파라미터 감소로 연산량 절감, 응답 속도 빨라짐

## 3. 작동 원리 (How it Works)
1. 입력 토큰이 게이팅 함수에 전달됨
2. 게이팅 함수(소형 LM 포함 가능)가 입력의 의미적 특성 분석
3. 전문가 리스트에서 Top-K 전문가 선택 (예: "수학 전문가", "코딩 전문가")
4. 선택된 전문가만 활성화되어 해당 입력 처리
5. 전문가 출력을 가중 합산하여 최종 출력 생성

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: GPT-4 (추정) — 수학, 코딩, 언어 등 도메인별 전문가로 구성된 대형 MoE 모델
- 사례 2: Mixtral 8×7B — 오픈소스 MoE 모델, 7B 전문가 8개로 56B 수준 성능을 30B 수준 연산으로 구현
- 사례 3: 경량 sLLM 설계 — 소형 모델에서 MoE 적용으로 특정 도메인 성능 향상

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 총 파라미터 대비 실제 연산량(활성 파라미터) 감소 → 빠른 추론
- 전문가별 특화 학습으로 다양한 도메인에서 높은 성능
- 모델 규모 확장 시 연산 비용 증가가 완만
### 단점 (Cons)
- 전체 파라미터가 메모리에 상주해야 하므로 RAM/VRAM 절감 효과 없음
- 게이팅 함수의 로드 밸런싱 실패 시 특정 전문가에 부하 집중 (전문가 붕괴 문제)
- 분산 학습 시 전문가 간 통신 오버헤드 발생
