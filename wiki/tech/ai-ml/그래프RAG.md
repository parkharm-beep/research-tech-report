# 그래프RAG (GraphRAG)

## 1. 개요 (Overview)
- **기술명:** 그래프RAG (GraphRAG)
- **정의:** 지식 그래프(Knowledge Graph)의 구조화된 정보와 검색 증강 생성(RAG)의 생성 능력을 결합하여, 데이터 간의 복잡한 관계를 파악하고 보다 정확한 답변을 생성하는 기술입니다.
- **배경:** 기존의 벡터 검색 기반 RAG는 개별 텍스트 청크의 유사도에만 의존하기 때문에, 문서 전체를 관통하는 핵심 주제를 파악하거나 여러 엔티티 간의 연결된 정보를 추론하는 데 한계가 있었습니다. 이를 해결하기 위해 데이터의 관계망을 활용하는 방식이 등장했습니다.

## 2. 주요 특징 (Key Features)
- **관계 중심 검색:** 단순 키워드/벡터 유사도를 넘어 엔티티 간의 연결성(Edge)을 바탕으로 정보를 탐색합니다.
- **계층적 요약:** 데이터를 커뮤니티 단위로 그룹화하고 요약하여, 세부적인 질문부터 포괄적인 질문까지 대응 가능합니다.
- **추론 능력 향상:** 지식 그래프의 구조를 활용해 직접적으로 연결되지 않은 정보 간의 연관성을 파악하는 '멀티홉(Multi-hop)' 추론에 강점이 있습니다.

## 3. 작동 원리 (How it Works)
1. **그래프 구축:** 비정형 텍스트에서 엔티티(노드)와 그들 사이의 관계(엣지)를 추출하여 지식 그래프를 생성합니다.
2. **커뮤니티 탐지:** 추출된 그래프에 알고리즘을 적용하여 밀접하게 연관된 노드들을 그룹화(Community)하고 각 그룹에 대한 요약문을 생성합니다.
3. **질의 처리:** 사용자의 질문이 들어오면 관련 있는 그래프 노드나 커뮤니티 요약을 검색합니다.
4. **답변 생성:** 검색된 구조화된 컨텍스트를 LLM에 전달하여 최종 답변을 생성합니다.

## 4. 주요 활용 사례 (Use Cases)
- **사례 1: 복합 문서 분석** - 수만 페이지에 달하는 보고서나 기술 문서에서 여러 개념이 얽힌 복잡한 질문에 대한 답변 추출.
- **사례 2: 부정 적발 및 수사** - 금융 거래나 통신 기록 등에서 엔티티 간의 의심스러운 관계망을 추적하고 요약.

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 전역적인 맥락 파악 능력이 뛰어나 "이 데이터셋의 주요 테마는 무엇인가?"와 같은 질문에 잘 답변합니다.
- 답변의 근거가 되는 엔티티와 관계가 명확하여 할루시네이션(환각)을 줄일 수 있습니다.
### 단점 (Cons)
- 지식 그래프를 구축하고 인덱싱하는 과정에서 LLM 토큰 소모량이 많고 비용이 높습니다.
- 데이터가 빈번하게 업데이트되는 환경에서 그래프를 실시간으로 갱신하는 것이 기술적으로 복잡합니다.
