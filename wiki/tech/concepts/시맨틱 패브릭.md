# 시맨틱 패브릭 (Semantic Fabric)

## 1. 개요 (Overview)
- **기술명:** 시맨틱 패브릭 / 시맨틱 데이터 패브릭 (Semantic Fabric / Semantic Data Fabric)
- **정의:** 멀티 에이전트 시스템에서 AI 에이전트들이 서로 지식을 의미적으로 공유하고 소통할 수 있도록 온톨로지 기반으로 데이터를 직조(weave)한 의미 기반 데이터 레이어
- **배경:** 단일 LLM 호출 수준을 넘어 수십 개의 에이전트가 업무 프로세스를 자동으로 분담하는 멀티 에이전트 시스템에서는 에이전트끼리 지식을 전달할 공통 언어·체계가 필요하다. 기존 엑셀·ERP 등 테이블 구조(투플)는 AI가 의미를 파악하기 어려워 온톨로지 기반의 의미 통합 레이어가 대안으로 부상

## 2. 주요 특징 (Key Features)
- [[온톨로지]]의 트리플(Triple) 구조로 이기종 데이터를 의미적으로 통합
- 에이전트 간 지식 전달에 공통 스키마·언어 제공 (RDF/OWL 등)
- 실시간 온톨로지 자동 생성(LLM이 텍스트에서 개체·관계 추출)과 시맨틱 패브릭 갱신의 상호 보완
- 분산된 수천 개의 데이터베이스를 '하늘에서 전쟁터를 내려다보듯' 조망 가능하게 함

## 3. 작동 원리 (How it Works)
- 이기종 데이터(ERP, 센서, 문서, API 등)를 온톨로지 스키마 기반의 트리플로 변환
- 변환된 데이터가 그래프 구조로 연결되어 시맨틱 패브릭 형성
- 에이전트 A가 작업 결과를 시맨틱 패브릭에 기록 → 에이전트 B가 의미 기반으로 읽어 다음 작업 수행
- LLM이 텍스트·머신 데이터를 읽고 새로운 트리플을 추출하여 패브릭을 실시간 갱신

비유: 모차르트가 오케스트라 전체 음악을 머릿속에 가지고 있지만 각 악기에 악보(공통 언어)를 주듯, 시맨틱 패브릭은 에이전트에게 공통 지식 언어를 제공

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: 제조 멀티 에이전트 — 구매·생산·인사·영업 데이터를 시맨틱 패브릭으로 통합, 에이전트 간 실시간 의사결정 지원
- 사례 2: 국방 작전 지휘 — 위성사진·휴민트·전자전 데이터를 온톨로지로 통합하여 지휘 에이전트에 공급 (팔란티어 JDC2 구조)
- 사례 3: 의료 진단 보조 — 증상·검사 결과·병력을 SNOMED 기반 시맨틱 패브릭으로 연결하여 진단 에이전트 지원

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 수천 개 이기종 DB를 의미적으로 통합 → 전사 데이터 빅픽처 확보
- 에이전트 간 지식 전달의 정확도·설명 가능성 향상
- LLM이 읽기 어려운 머신 데이터(시계열·수치)를 온톨로지화하여 처리 가능하게 함

### 단점 (Cons)
- 도메인 전문가(온톨로지스트)와 엔지니어의 협업이 구축에 필수
- 초기 스키마 설계 품질이 전체 패브릭의 추론 신뢰도를 결정
- 아직 성숙한 플랫폼·표준화가 진행 중인 신흥 영역
