# 디지털 스레드 (Digital Thread)

## 1. 개요 (Overview)
- **기술명:** 디지털 스레드 (Digital Thread)
- **정의:** 제품의 설계→제조→운영→폐기까지 전체 수명주기에 걸쳐 생성되는 디지털 데이터를 단일 연속 흐름으로 연결하는 개념
- **배경:** 제조 산업에서 각 단계(CAD, PLM, ERP, MES, IoT)의 데이터가 사일로에 갇혀 연계가 어렵다는 문제 해결을 위해 등장했다. [[디지털트윈]]과 짝을 이루는 개념으로, 디지털 스레드는 데이터 흐름, 디지털 트윈은 그 결과물의 모델이다.

## 2. 주요 특징 (Key Features)
- **종단간 데이터 연속성:** 설계 파라미터가 제조→운영 데이터와 추적 가능하게 연결
- **단일 진실 공급원 (Single Source of Truth):** 모든 수명주기 데이터를 하나의 참조점에서 접근
- **변경 이력 추적:** 어느 단계에서 무엇이 변경됐는지 완전한 이력 보존
- **자동화 연계:** 설계 변경이 제조 공정·품질 기준에 자동 반영

## 3. 작동 원리 (How it Works)
- PLM(Product Lifecycle Management) 플랫폼이 설계 데이터를 중앙 저장
- 제조 단계의 ERP·MES와 API로 실시간 연동
- 운영 단계의 IoT 센서 데이터가 디지털 스레드에 추가 누적
- AI·분석 도구가 전체 데이터 흐름을 분석하여 인사이트 도출

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: 항공우주 — 부품 번호→설계 스펙→제조 기록→MRO 이력을 하나의 스레드로 연결
- 사례 2: 방위산업 — 규제 추적을 위한 완전한 제품 이력 관리

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 수명주기 전체의 가시성 확보 → 품질 문제의 근본 원인 추적 용이
- 단계 간 데이터 재입력·오류 제거
### 단점 (Cons)
- 레거시 시스템 통합의 복잡성
- 조직 간 데이터 표준 합의 필요
