# 프레데터 봇 (Predator Bot)

## 1. 개요 (Overview)
- **기술명:** 프레데터 봇 (Predator Bot)
- **정의:** AI와 자동화 기술을 결합해 특정 대상·데이터를 선별·추적·공격적으로 활용하는 고도화된 자동화 공격 프로그램
- **배경:** AI 도입으로 봇이 정상 이용자 행동을 실시간으로 모방하고 대량 데이터 분석 기반 능동적 의사결정이 가능해지면서, 기존 정적 방어 체계(캡차·IP 차단)로는 탐지·차단이 어려운 새로운 위협으로 부상했다.

## 2. 주요 특징 (Key Features)
- **능동적 탐색:** 단순 반복 작업을 수행하는 일반 봇과 달리, 대량 데이터 분석 + 실시간 의사결정으로 목표를 스스로 탐색
- **인간 행동 모방:** 정상 이용자처럼 행동 패턴을 흉내 내며 실시간으로 공격 방식 변경
- **AI 기반 적응:** 기존 방어 탐지 시 전략을 동적으로 수정 → 정적 규칙 기반 방어 우회

## 3. 공격 경로 및 위협 유형

| 공격 유형 | 설명 |
|---|---|
| **API 공격** | 로그인·결제·장바구니 등 서비스 핵심 API를 주요 경로로 삼아 로직 우회 |
| **계정 탈취 (ATO)** | 크레덴셜 스터핑 등으로 사용자 계정 무단 접근 |
| **데이터 스크래핑** | 가격 정보·개인정보 등 대량 데이터 자동 수집 |
| **상품·티켓 선점** | 한정 상품·공연 티켓을 자동 구매하여 재판매 목적으로 선점 |
| **거래 조작** | 가격 조작, 재고 허위 소진 등 비즈니스 로직 악용 |
| **허위 계정 생성** | 대규모 소셜 계정·리뷰 계정 자동 생성 |

## 4. 기존 방어 기법의 한계
- **CAPTCHA:** AI 기반 이미지 인식으로 우회 가능
- **IP 차단:** 주거용 프록시(Residential Proxy)·분산 IP 활용으로 회피
- **정적 규칙 기반 WAF:** 정상 트래픽과 구별 불가한 행동 패턴으로 탐지 불가
- **Rate Limiting:** 요청 속도를 정상 사용자 수준으로 조절하여 우회

## 5. 대응 방안 (How it Works — Defense)
보안 업계 권고 4단계 대응 체계:
1. **전체 API 식별 및 관리** — 섀도우 API 포함 전체 엔드포인트 인벤토리화
2. **행위 기반 봇 탐지** — 마우스 움직임, 클릭 패턴, 세션 흐름 등 행동 분석으로 이상 징후 감지
3. **다중인증(MFA) 도입** — 계정 탈취 성공 시에도 접근 차단
4. **비즈니스 로직 모니터링** — 정상 업무 흐름 이탈 패턴(예: 비정상적 구매 속도) 실시간 감시

## 6. 장단점 (Pros & Cons)
### 공격자 관점의 강점
- AI 적응으로 방어 업데이트에 자동 대응
- 대규모 병렬 공격으로 인건비 없이 수익 창출
- 정상 트래픽과의 구별이 어려워 탐지 회피 용이
### 방어자 관점의 과제
- 행위 기반 탐지는 오탐(False Positive) 위험 → 정상 사용자 차단 가능성
- 봇과 정상 트래픽 구별을 위한 고비용 머신러닝 모델 필요
- API 전체 가시성 확보가 선결 과제
