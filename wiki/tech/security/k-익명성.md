# k-익명성 (k-Anonymity)

## 1. 개요 (Overview)
- **기술명:** k-익명성 (k-Anonymity)
- **정의:** 데이터셋 내 어떤 개인도 최소 k명의 다른 레코드와 구별되지 않도록 하여 재식별 위험을 방지하는 개인정보 보호 기법
- **배경:** AI 모델 학습용 공공데이터를 공유할 때 개인 식별 위험을 제거하면서도 데이터 유용성을 최대한 유지해야 하는 요구에서 등장

## 2. 주요 특징 (Key Features)
- k값이 클수록 익명성 보장 수준이 높고, 데이터 유용성은 낮아지는 트레이드오프
- 준식별자(Quasi-Identifier: 나이, 성별, 우편번호 등) 조합으로 특정인 식별 불가 조건 부여
- 「개인정보 보호법」 및 행안부 「가명정보 처리 가이드라인(2026.3)」의 핵심 기법

## 3. 작동 원리 (How it Works)
가명정보 처리 5단계 중 3단계(가명처리)에서 적용:
1. **사전준비** — 대상 데이터·처리 목적·책임자 확인
2. **위험성 검토** — 준식별자 조합으로 재식별 가능성 평가
3. **가명처리** — k-익명성 확보 (일반화·마스킹·삭제 등 적용)
4. **적정성 검토** — 처리 결과의 재식별 위험 재평가
5. **안전한 관리** — 처리된 데이터의 접근 통제·감사 추적

주요 처리 기법:
- **일반화(Generalization)**: 정밀한 값을 범위로 대체 (예: 나이 32 → 30대)
- **마스킹(Masking)**: 일부 값을 `*`로 치환 (예: 주민번호 뒷자리 마스킹)
- **억제(Suppression)**: 재식별 위험이 높은 레코드 전체 삭제

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: 공공 의료데이터 — k-익명성 처리 후 AI 모델 학습 데이터로 제공
- 사례 2: 행정 통계 — 지역·연령·성별 조합 재식별 방지 후 공개

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 법·제도적 요건(개인정보 보호법)을 충족하는 공식 기법
- 데이터 유용성을 어느 정도 유지하면서 프라이버시 보호 가능

### 단점 (Cons)
- k값 증가 시 데이터 유용성 저하 — AI 모델 품질에 영향
- **동질성 공격(Homogeneity Attack)**: 같은 그룹 내 민감 속성이 동일하면 여전히 유추 가능
- **배경지식 공격(Background Knowledge Attack)**: 외부 정보와 결합 시 재식별 가능
