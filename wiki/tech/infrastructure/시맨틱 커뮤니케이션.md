# 시맨틱 커뮤니케이션 (Semantic Communication)

## 1. 개요 (Overview)
- **기술명:** 시맨틱 커뮤니케이션 (Semantic Communication, 의미 기반 통신)
- **정의:** 비트(bit) 단위의 정확한 데이터 복원이 아닌 메시지의 **의미(Meaning)와 의도(Intent)** 전달에 집중하는 차세대 통신 패러다임
- **배경:** 클로드 섀넌(1948)의 정보 이론은 메시지의 정확한 복원에만 집중했다. 6G 시대 데이터 트래픽 폭증으로 물리적 대역폭이 **섀넌 한계(Shannon Limit)** 에 근접하면서, 보내는 내용 자체를 최소화하는 새로운 접근이 필요해졌다.

## 2. 주요 특징 (Key Features)
- 원본 데이터 대신 **의미적으로 압축된 표현**만 전송
- 텍스트 전송 시 기존 방식 대비 **100배 이상** 압축 효율
- 열악한 채널 환경에서도 의미 전달 보장 (Graceful Degradation)
- 송·수신 측이 **공유 지식 베이스(Shared Knowledge Base)** 보유 전제
- AI(딥러닝)를 통신 시스템의 핵심 구성 요소로 통합

## 3. 핵심 기술

### Deep JSCC (Deep Joint Source-Channel Coding, 결합 소스-채널 코딩)
- 기존: 데이터 압축(소스 코딩) + 오류 정정(채널 코딩)을 분리 처리
- Deep JSCC: 딥러닝 신경망 하나로 두 과정을 **동시에 최적화**
- **절벽 효과(Cliff Effect) 방지:** 채널 상태 악화 시 기존 방식은 성능이 급격히 붕괴하지만, Deep JSCC는 완만하게 저하되며 최소한의 의미를 유지
- 이미지·음성·텍스트 등 모달리티별 전용 신경망 설계 가능

### 공유 지식 베이스 (Shared Knowledge Base)
- 송신자·수신자가 사전 학습된 AI 모델 또는 지식 그래프를 공유
- 수신자는 압축된 의미 정보를 받아 지식 베이스로 복원
- 예: "낙상 감지" 신호만 전송 → 수신 측에서 원본 영상 없이도 상황 파악

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: 자율주행 — 모든 센서 데이터 전송 대신 "전방 30m 사고 발생" 의미 정보만 교환 → 초저지연 안전 반응
- 사례 2: 원격 의료 — 환자 영상 원본 대신 "낙상 감지" 추론 결과만 전송 → 대역폭 절약 + 프라이버시 보호
- 사례 3: 홀로그램 미디어 — 시맨틱 정보만 전송, 수신단 생성형 AI가 배경 장면 복원 → 홀로그램을 현실적인 대역폭으로 구현
- 사례 4: IoT 센서망 — 수억 개 센서가 원본 대신 의미 요약만 전송 → 네트워크 혼잡 해소

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 대역폭 효율 획기적 향상 (100배 이상 압축)
- 불안정한 채널에서도 의미 전달 보장
- 민감 데이터(영상, 음성) 원본 비전송 → 프라이버시 보호 효과
### 단점 (Cons)
- 이종 기기 간 지식 베이스 동기화 표준 미성숙
- AI 환각이 의미 해석 오류로 이어질 경우 치명적 상황 가능 (자율주행, 의료)
- 지식 베이스 유지·업데이트 비용
- 섀넌 이론 기반 기존 시스템과의 호환성 문제
