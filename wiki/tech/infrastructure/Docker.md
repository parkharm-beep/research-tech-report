# Docker (도커)

## 1. 개요 (Overview)
- **기술명:** Docker (도커)
- **정의:** 애플리케이션과 그 의존성을 컨테이너라는 격리된 실행 환경으로 패키징하여, 어느 환경에서든 동일하게 실행할 수 있게 하는 플랫폼
- **배경:** "내 컴퓨터에서는 됐는데 서버에서는 안 된다"는 환경 불일치 문제를 해결하기 위해 등장. AI 바이브코딩 워크플로우에서 로컬 개발 환경 구성과 배포에 필수적으로 활용된다.

## 2. 주요 특징 (Key Features)
- **컨테이너 격리:** 각 애플리케이션이 독립적인 실행 환경(컨테이너)에서 동작
- **이식성:** 로컬·서버·클라우드 어디서든 동일한 환경 재현 가능
- **이미지 기반:** Dockerfile로 환경 정의 후 이미지를 빌드하여 배포
- **경량화:** VM보다 가볍고 빠른 기동 시간

## 3. 작동 원리 (How it Works)
- **Dockerfile:** 애플리케이션 실행에 필요한 환경, 의존성, 명령어를 정의하는 파일
- **Image (이미지):** Dockerfile을 빌드한 결과물 (실행 가능한 스냅샷)
- **Container (컨테이너):** 이미지를 실행한 인스턴스 — 실제로 동작하는 프로세스
- **Docker Hub:** 공개 이미지 레지스트리 — 다른 사람이 만든 이미지를 가져다 쓸 수 있음

터미널 기본 명령:
```bash
docker pull {이미지명}       # 이미지 다운로드
docker run {이미지명}        # 컨테이너 실행
docker ps                    # 실행 중인 컨테이너 목록
docker stop {컨테이너ID}     # 컨테이너 중지
```

## 4. 주요 활용 사례 (Use Cases)
- 사례 1: 바이브코딩으로 만든 웹 앱을 로컬에서 실행·테스트
- 사례 2: 팀원들과 동일한 개발 환경 공유 (Node.js 버전 불일치 등 해소)
- 사례 3: CI/CD 파이프라인에서 테스트·배포 자동화
- 사례 4: Kubernetes(쿠버네티스)와 연동하여 대규모 컨테이너 오케스트레이션

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 환경 불일치 문제 해소
- 빠른 환경 설정 및 재현
- 마이크로서비스 아키텍처와 궁합 우수
### 단점 (Cons)
- 비개발자에게는 개념과 명령어가 낯설어 초기 학습 비용 존재
- Windows 환경에서 일부 호환성 이슈 (WSL2 필요)
- 이미지 빌드·레이어 관리 지식이 없으면 이미지가 비대해질 수 있음
