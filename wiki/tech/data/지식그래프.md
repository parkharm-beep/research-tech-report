# 지식그래프 (Knowledge Graph)

## 1. 개요 (Overview)
- **기술명:** 지식그래프 (Knowledge Graph)
- **정의:** 현실 세계의 객체(Entity), 개념, 사건 간의 관계를 노드(Node)와 엣지(Edge)로 연결된 네트워크 형태로 구조화하여 기계가 이해할 수 있도록 만든 지식 베이스입니다.
- **배경:** 단순한 키워드 매칭 중심의 검색 한계를 극복하고, 데이터 간의 맥락(Context)과 의미(Semantics)를 파악하여 "문자열이 아닌 사물(Things, Not Strings)" 관점에서 정보를 처리하기 위해 등장했습니다. 2012년 구글이 검색 엔진에 도입하며 대중화되었습니다.

## 2. 주요 특징 (Key Features)
- **의미론적 연결성:** 데이터를 단순 저장이 아닌 의미적 관계(Triple: 주어-술어-목적어)로 연결하여 풍부한 맥락을 제공합니다.
- **유연한 스키마:** 관계형 DB와 달리 새로운 개체나 관계 유형을 추가하기 쉬운 유연한 구조를 가집니다.
- **논리적 추론:** 명시되지 않은 관계를 온톨로지 규칙이나 연결망을 통해 새롭게 도출(Inference)할 수 있습니다.
- **다중 홉(Multi-hop) 탐색:** 복잡하게 얽힌 관계를 그래프 순회 알고리즘을 통해 효율적으로 탐색합니다.

## 3. 작동 원리 (How it Works)
1. **데이터 수집:** 정형/비정형 소스로부터 데이터를 수집합니다.
2. **정보 추출:** NLP 기술(개체명 인식, 관계 추출 등)을 사용해 텍스트에서 삼중항(Triple)을 뽑아냅니다.
3. **지식 통합:** 동일 개체를 식별(Entity Linking)하고 중복을 제거하여 정제된 지식망을 구축합니다.
4. **저장 및 관리:** 그래프 데이터베이스(Neo4j 등) 또는 트리플스토어에 저장합니다.
5. **활용 및 추론:** SPARQL/Cypher 쿼리나 추론 엔진을 통해 인사이트를 도출합니다.

## 4. 주요 활용 사례 (Use Cases)
- **의미 기반 검색:** 구글 지식 패널처럼 검색어의 배경 지식과 연관 정보를 함께 제공합니다.
- **GraphRAG:** LLM과 결합하여 환각 현상을 줄이고 정확한 근거에 기반한 답변을 생성합니다.
- **사기 탐지 (FDS):** 금융 거래 네트워크를 분석하여 자금 세탁이나 이상 거래 패턴을 식별합니다.
- **개인화 추천:** 사용자의 취향과 아이템 속성 간의 복잡한 관계를 분석해 정교한 추천을 제공합니다.

## 5. 장단점 (Pros & Cons)
### 장점 (Pros)
- 데이터 간의 복잡한 관계를 직관적으로 표현하고 분석 가능합니다.
- 설명 가능한 AI(xAI) 구현을 위한 논리적 근거를 제공합니다.
- 다양한 출처의 데이터를 의미론적으로 통합하기에 유리합니다.
### 단점 (Cons)
- 초기 구축 시 온톨로지 설계 및 데이터 정제 비용이 높습니다.
- 데이터 규모가 커질수록 그래프 순회 연산에 많은 컴퓨팅 자원이 소모됩니다.
- 지속적인 지식 업데이트와 품질 관리가 까다롭습니다.
